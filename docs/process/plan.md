# 实时对战枪战游戏 — 执行计划

## 项目定义
- **名称**：枪火竞技场（Gun Arena）
- **形态**：2D 俯视角竞技场枪战，浏览器打开即玩
- **对战**：玩家 vs 多个 AI 机器人实时对战（移动、索敌、射击、走位均有 AI 决策）
- **风格**：卡通休闲，明亮配色，清晰可读性
- **技术**：React + TypeScript + Canvas 渲染（游戏循环），Tailwind 做 UI 外壳
- **交付**：网页版本（website_version_manager build_version）

## 核心玩法
- WASD 移动 + 鼠标瞄准射击（支持移动端虚拟摇杆可选）
- 竞技场地图：掩体障碍、出生点、道具刷新点（回血包、武器箱、护盾）
- 多种武器：手枪 / 霰弹枪 / 步枪 / 狙击枪（伤害、射速、弹匣、散射不同）
- AI 机器人：状态机（巡逻/追击/撤退/抢道具），不同难度个性
- 计分：击杀数 + 存活，回合制（如 3 分钟一局或先到 N 杀）
- 命中反馈：粒子特效、屏幕震动、击杀播报、音效（WebAudio 合成，无外部资源依赖）

## 阶段规划

### Stage 1 — 技能加载与脚手架
- 加载 `vibecoding-webapp-swarm`（浏览器游戏属 web UI 任务，设计先行的 React 工作流）
- 按 skill 指引搭建项目骨架

### Stage 2 — 游戏核心实现（子代理分工）
- 游戏引擎层：游戏循环、输入系统、碰撞检测、相机
- 实体与战斗：玩家/机器人/子弹/武器系统/伤害结算
- AI 决策：机器人行为状态机与难度梯度
- 地图与道具：竞技场布局、掩体、道具刷新
- UI/HUD：血量、弹药、小地图、击杀播报、结算画面
- 音效与特效：WebAudio 合成音效 + 粒子系统

### Stage 3 — 验证与打磨
- 构建通过、运行无错
- 可玩性检查：帧率、操作手感、AI 强度曲线
- 视觉走查：卡通风格一致性、信息可读性

### Stage 4 — 交付
- `website_version_manager` build_version 保存网页版本，输出预览地址

## 里程碑验收
1. 打开页面即可开始一局 3 分钟对战
2. 至少 4 种武器 + 3 种道具 + 5 个 AI 机器人同场
3. 完整的开始 → 对战 → 结算 → 再来一局闭环
